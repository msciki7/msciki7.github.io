회원가입(Signup)은 인증(Authentication)이 아니다. 회원가입은 단지 새로운 User 데이터를 생성하여 DB에 저장하는 과정이다.

- 회원가입: 계정 생성
- 로그인: 계정 확인(인증)


## 회원가입 흐름 전체 요약

회원가입 요청이 들어오면 다음과 같은 순서로 처리된다.

이 단계에서는 Spring Security나 JWT는 개입하지 않는다.



## Controller의 역할 — 요청 수신

Controller의 책임은 명확

- HTTP 요청을 받는다.
- 요청 데이터를 Service로 전달한다.
- 응답을 반환한다.


## SignupRequest — 입력 검증은 어디서 하는가

회원가입 요청에서의 형식 검증은 DTO에서 처리한다.

- 필수 값 여부
- 이메일 형식
- 전화번호 형식
Controller나 Service에서 if문으로 검증하지 않는다.

- DTO→ 형식이 맞는 입력인가?
- Service → 비즈니스적으로 가능한가?


## UserService.signUp — 회원가입의 핵심 로직

이 메서드가 회원가입의 실질적인 중심



## 회원가입에서 수행하는 검증들



1. 중복 검사
- 이메일 중복
- 전화번호 중복
DB 조회 기반 비즈니스 검증이므로 Service 책임

1. 비밀번호 처리
회원가입에서 비밀번호는 다음 순서로 처리된다.

1. trim 처리(공백 제거)
1. null / empty 검사
1. 비밀번호 정책 검증
1. PasswordEncoder로 암호화
1. 암호화된 값만 저장


User 엔티티는 비밀번호를 검증하지 않는다. 

User는 암호화된 결과를 저장한다.



## 왜 회원가입에서는 토큰을 발급하지 않을까?

회원가입 성공 시 반환되는 것은 다음과 같은 정보들

회원가입은 계정 생성 단계이기 때문에 로그인 상태를 의미하는 토큰을 발급하지 않는다.

토큰 발급은 로그인에서 수행된다.
