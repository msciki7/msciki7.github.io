# 스택이란?

&ensp;LIFO 구조(Last In First Out) -> 나중에 들어온 것이 먼저 나간다.<br/>
&ensp;예<br/>
```c++
push(10)
push(20)
push(30)

pop() → 30
pop() → 20
```

&ensp;스택을 배열로 구현하는 핵심 아이디어<br/>
&ensp;배열:<br/>
```css
[ ][ ][ ][ ][ ]
```

&ensp;여기에 데이터를 쌓는다.<br/>
&ensp;그리고 우리는 하나의 변수가 필요하다.:<br/>
```cpp
top
```

&ensp;top이 의미하는 것<br/>
* top = -1 -> 비어있음
* top = 0 -> 첫 번째 원소
* top = n -> 마지막 원소 위치

&ensp;메모리 그림<br/>
&ensp;초기 상태<br/>
```cpp
stack: [ ][ ][ ][ ][ ]
top = -1
```

&ensp;push(10)
1. top++
2. stack[top] = 10

```cpp
stack: [10][ ][ ][ ][ ]
top = 0
```

&ensp;push(20)<br/>
```cpp
stack: [10][20][ ][ ][ ]
top = 1
```

&ensp;pop()<br/>
1. stack[top] 반환
2. top--

```cpp
stack: [10][20][ ][ ][ ]
top = 0
```

&ensp;실제 값은 남아있지만 top이 줄어들면 무시됨<br/>

# 배열 기반 스택 구현(c++)

&ensp;구조 정의<br/>

```cpp
#include <iosream>
using namespace std;

#define MAX_SIZE 100

struct Stack{
  int data[MAX_SIZE];
  int top;
};
```

&ensp;초기화<br/>
```cpp
void initStack(Stack* s){
  s->top = -1;
}
```

&ensp;비어있는지 확인<br/>
```cpp
bool isEmpty(Stack* s){
  return (s->top == -1)
}
```

&ensp;가득 찼는지 확인<br/>
```cpp
bool isFull(Stack *s){
  return (s->top == MAX_SIZE -1);
}
```

&ensp;push 연산<br/>
```cpp
void push(Stack* s, int value){
  if(isFull(s)){
    cout <<"스택이 가득 찼습니다." << endl;
    return;
  }

  s->top++;
  s->data[s->top] = value;
}
```

&ensp;pop 연산<br/>
```cpp
int pop(Stack* s){
  if(isEmpty(s)){
    cout << "스택이 비어있습니다." << endl;
    return -1;
  }

  int value = s->data[s->top];
  s->top--;
  return value;
}
```

&ensp;peek(맨 위 확인)<br/>
```cpp
int peek(Stack* s){
  if(isEmpty(s)){
    cout <<"스택이 비어있습니다." << endl;
    return -1;
  }

  return s->data[s->top];
}
```

&ensp;전체 테스트 코드<br/>
```cpp
int main(){
  Stack s;
  initStack(&s);

  push(&s, 10);
  push(&s, 20);
  push(&s, 30);

  cout << "TOP 값: " << peak(&s) << endl;  // 30

  cout << "Pop: " << pop(&s) << endl;     // 30
  cout << "Pop: " << pop(&s) << endl      // 20
  cout << "Pop: " << pop(&s) << endl;     // 10

  return 0;
}
```

## 시간 복잡도

| 연산   | 시간   |
| ---- | ---- |
| push | O(1) |
| pop  | O(1) |
| peek | O(1) |

&ensp;배열 기반 스택은 모든 연산이 빠름<br/>

## 배열 스택의 단점

* 크기 고정
* overflow 발생 가능

```
top == MAX_SIZE -1
```

# 연결 리스트 기반 스택

&ensp;연결 리스트 기반 스택은:<br/>
```
top = head 포인터
```

&ensp;스택의 맨 위(top)가 곧 연결 리스트의 head다.<br/>

&ensp;초기 상태<br/>
```cpp
top → NULL
```

&ensp;push(10)<br/>
```cpp
top → [10 | NULL]
```

&ensp;push(20)<br/>
&ensp;새 노드 생성 -> 앞에 붙임<br/>
```cpp
top → [20] → [10] → NULL
```

&ensp;push(30)<br/>
```cpp
top → [30] → [20] → [10] → NULL
```

&ensp;pop()<br/>
&ensp;맨 앞 제거<br/>
```cpp
top → [20] → [10] → NULL
```

&ensp;연결 리스트의 "맨 앞 삽입/삭제"가 곧 스택<br/>

# 연결 리스트 기반 스택이 좋을까?

&ensp;장점<br/>
* 크기 제한 없음
* overflow 없음 (메모리만 충분하면 됨)

&ensp;단점<br/>
* 포인터 사용
* 배열보다 메모리 조금 더 사용

# 연결 리스트 기반 스택 구현(c++)

&ensp;노드 정의<br/>
```cpp
#include <iostream>
using namespace std;

struct Node{
  int data;
  Node* next;
};
```

&ensp;스택 구조 정의<br/>
```cpp
struct Stack{
  Node* top;
}
```

&ensp;초기화<br/>
```cpp
void initStack(Stack* s){
  s->top = NULL;
}
```

&ensp;비어있는지 확인<br/>
```cpp
bool isEmpty(Stack* s){
  return (s->top == NULL);
}
```

&ensp;push 연산<br/>
```cpp
void push(Stack* s, int value){
  Node* newNode = new Node;
  newNode->data = value;

  newNode->next = s->top;   // 기존 top을 뒤로 밀기
  s->top = newNode;         // top을 새 노드로 변경
}
```

```css
newNode → 기존 top
top → newNode
```

&ensp;pop 연산<br/>
```cpp
int pop(Stack* s){
  if(isEmpty(s)){
    cout << "스택이 비어있습니다." << endl;
    return -1;
  }

  Node* temp = s->top;
  int value = temp->data;

  s->top = temp->next;  // 다음 노드를 top으로
  delete temp;          // 메모리 해제

  return value;
}
```

&ensp;peek<br/>
```cpp
int peek(Stack* s){
  if(isEmpty(s)){
    cout << "스택이 비어있습니다." << endl;
    return -1;
  }

  return s->top->data;
}
```

&ensp;전체 테스트 코드<br/>
```cpp
int main(){
  Stack s;
  initStack(&s);

  push(&s, 10);
  push(&s, 20);
  push(&s, 30);

  cout << "Top 값: " << peek(&s) << endl;   // 30

  cout << "Pop: " << pop(&s) << endl;      // 30
  cout << "Pop: " << pop(&s) << endl;      // 20
  cout <<"Pop: " << pop(&s)> << endl;      // 10

  return 0;
}
```

&ensp;포인터 흐름 완전 정리<br/>
&ensp;push 할 때:<br/>
```css
newNode->next = top
top = newNode
```

&ensp;pop 할 때:<br/>
```css
temp = top
top = top->next
delete temp
```

# 배열 스택 vs 연결 리스트 스택 비교

| 항목     | 배열  | 연결 리스트 |
| ------ | --- | ------ |
| 크기     | 고정  | 동적     |
| 구현 난이도 | 쉬움  | 포인터 필요 |
| 속도     | 빠름  | 거의 동일  |
| 메모리    | 효율적 | 포인터 추가 |
