1. 자료 구조 개념 정리
=====

&ensbr/>
자료 구조 = 데이터를 효율적으로 저장하고 관리하는 방법<br/>
* 데이터를 어떤 규칙으로 정리해두는 방식
* 메모리를 효율적으로 사용하고 빠르게 검색, 추가, 삭제하기 위해 필요함

&ensp;자료구조에 따라 작업 속도가 달라짐<br/>

&ensp;자료구조와 알고리즘 관계
-----

&ensp;알고리즘: 문제를 해결하기 위한 절차 또는 방법

&ensp;관계: `자료구조 -> 사용할 수 있는 알고리즘 결정` <br/>

&ensp;ex<br/>
* 정렬된 배열 -> 이진 탐색 가능
* 연결 리스트 -> 순차 탐색

&ensp;`자료 구조 + 알고리즘 = 프로그램 성능` <br/>

2. 자료 구조가 중요한 이유
====

&ensp;같은 기능을 하는 프로그램이라도 자료구조에 따라 성능이 크게 달라진다.<br/>

| 구조     | 검색 속도 |
| ------ | ----- |
| 배열     | O(n)  |
| 해시 테이블 | O(1)  |

&ensp;-> 자료구조 선택이 성능을 결정한다.<br/>

3. 자료구조의 큰 분류
=====

&ensp;1) 선형 자료구조<br/>
&ensp;데이터가 한 줄로 연결된 구조<br/>
* 배열(Array)
* 연결 리스트(Linked List)
* 스택(Stack)
* 큐(Queue)

&ensp;2) 비선형 자료구조<br/>
&ensp;데이터가 계층 또는 네트워크 형태<br/>
* 트리 (Tree)
* 그래프 (Graph)
* 힙(Heap)
* 해시 테이블 (Hash Table)

4. 핵심 알고리즘 8가지
====

&ensp;1. 배열(Array)<br/ㄴ
&ensp;같은 타입의 데이터를 연속된 메모리 공간에 저장하는 구조<br/>

&ensp;예<br/>
```makefile
index:   0   1   2   3
value:  10  20  30  40
```

&ensp;특징<br/>
* 인덱스로 바로 접근 가능
* 메모리에 연속 저장

&ensp;시간 복잡도
* 접근: O(1)
* 삽입/삭제: O(n)

&ensp;장점<br/>
* 빠른 접근 속도

&ensp;단점<br/>
* 크기 변경 어려움
* 중간 삽입/삭제 느림

&ensp;2. 연결 리스트(Linked List)
&ensp;데이터와 다음 노드의 주소를 저장하는 구조<br/>

&ensp;구조<br/>
```css
[10 | next] → [20 | next] → [30 | next]
```

&ensp;특징<br/>
* 메모리에 연속적으로 저장되지 않음
* 포인터로 연결됨

&ensp;시간 복잡도<br/>
* 접근: O(n)
* 삽입/삭제: O(1) (위치만 알면)

&ensp;장점<br/>
* 크기 제한 없음
* 삽입/삭제 빠름

&ensp;단점<br/>
* 탐색 느림
* 포인터 저장 공간 필요

&ensp;3. 스텍(Stack)<br/>
&ensp;-> LIFO (Last In First Out)<br/>

&ensp;예<br/>

```scss
push(1)
push(2)
push(3)

pop() → 3
```

&ensp;주요 연산<br/>
* push: 데이터 넣기
* pop: 데이터 꺼내기
* peek: 맨 위 확인

&ensp;사용 예<br/>
* 함수 호출
* 되돌리기(Undo)
* 괄호 검사

&ensp;4. 큐(Queue)<br/>
&ensp;먼저 들어온 데이터가 먼저 나가는 구조 -> FIFO(First In First Out)<br/>

&ensp;예<br/>
```scss
enqueue(1)
enqueue(2)
enqueue(3)

dequeue() → 1
```

&ensp;주요 연산<br/>
* enqueue: 삽입
* dequeue: 삭제

&ensp;사용 예<br/>
* 대기열
* 프린터 작업
* BFS 탐색

&ensp;5. 해시 테이블(Hash Table)<br/>
&ensp;개념<br/>
&ensp;키(key)를 해시 함수로 변환해서 빠르게 데이터를 찾는 구조<br/>

&ensp;구조<br/>
```
key → hash 함수 → index
```

&ensp;예<br/>

&ensp;특징<br/>
* 평균 검색 속도: O(1)

&ensp;문제<br/>
* 해시 충돌 발생 가능

&ensp;해결 방법<br/>
* 체이닝
* 개방 주소법

&ensp;사용 예<br/>
* 로그인 정보 저장
* 사전(Dicrionary)
* 캐시 시스템

&ensp;6. 트리(Tree)<br/>
&ensp;계층 구조를 가진 자료구조<br/>

&ensp;구조 예<br/>

```css
      A
     / \
    B   C
   /
  D
```

&ensp;용어<br/>
* 루트: 가장 위 노드
* 부모: 위 노드
* 자식: 아래 노드
* 리프: 자식 업슨ㄴ 노드

&ensp;사용 예<br/>
* 파일 시스템
* 조직도
* 데이터 검색

&ensp;7. 힙(Heap)
&ensp;완전 이진 트리 구조<br/>

&ensp;종류<br/>
&ensp;최소 힙<br/>
&ensp;부모 ≤ 자식<br/>
```makedown
      1
     / \
    3   5
```

&ensp;최대 힙<br/>
&ensp;부모 ≥ 자식<br/>
```makedown
      9
     / \
    6   4
```

&ensp;특징<br/>
* 우선순위 큐 구현에 사용
* 루트가 항상 최대값 또는 최솟값

&ensp;시간 복잡도<br/>
* 삽입: O(log n)
* 삭제: O(log n)

&ensp;8. 그래프(Graph)<br/>
&ensp;노드와 간선으로 이루어진 구조<br/>

&ensp;예<br/>
```less
A — B
|   |
C — D
```

&ensp;특징<br/>
* 복잡한 관계 표현 가능

&ensp;사용 예<br/>
* 지도 경로 탐색
* SNS 친구 관계
* 네트워크 구조

5. 전체 구조 한눈에 정리
====

```markdown
자료구조
│
├── 선형 구조
│   ├── 배열
│   ├── 연결 리스트
│   ├── 스택
│   └── 큐
│
└── 비선형 구조
    ├── 트리
    ├── 힙
    ├── 그래프
    └── 해시 테이블
```
