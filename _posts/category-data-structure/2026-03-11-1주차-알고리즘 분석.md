# 알고리즘과 데이터구조

&ensp;알고리아이디br/>
알고리즘(Algorithm)<br/>
&ensp;어떤 문제를 유한한 시간 안에 해결하는 단계적 절차<br/>
&ensp;예<br/>
* 배열에서 최대값 찾기
* 정렬하기
* 그래프에서 최단경로 찾기

&ensp;데이터 구조(Data Structure)<br/>
&ensp;데이터를 효율으로 저장하고 접근하는 방법<br/>
&ensp;예<br/>
* 배열(Araay)
* 스택(Stack)
* 큐(Queue)
* 트리(Tree)
* 그래프(Graph)

&ensp;알고리즘 + 데이터 구조 = 프로그램의 성능 결정<br/>

## 좋은 알고리즘의 기준

&ensp;좋은 알고리즘은 두 가지로 평가한다.<br/>
&ensp;1. 실행시간(Running Time)<br/>
&ensp;얼마나 빨리 수행되는가<br/>

&ensp;2. 메모리 사용량<br/>
&ensp;얼마나 적은 메모리를 사용하는가<br/>

# 실행시간(Running Time)

&ensp;대부분 알고리즘은<br/>
&ensp;입력 -> 처리 -> 출력<br/>
&ensp;구조를 가진다.<br/>
&ensp;그리고 입력 크기 n이 커질수록 실행시간이 증가한다.<br/>

| 입력 크기         | 실행시간  |
| ------------- | ----- |
| n = 10        | 빠름    |
| n = 1000      | 느림    |
| n = 1,000,000 | 매우 느림 |

&ensp;그래서 알고리즘 분석에는 입력 크기 n에 따라 실행시간이 어떻게 증가하는지 본다.<br/>

# 알고리즘 실행시간 분석 방법

&ensp;1. 실험적 방법<br/>
1. 프로그램 구현
2. 다양한 입력으로 실행
3. 실제 시간 측정

&ensp;문제점<br/>
* 컴퓨터 성능 영향
* OS 영향
* 컴파일러 영향
* 구현 난이도

&ensp;그래서 실제 알고리즘 분석에서는 잘 사용하지 않는다.<br/>

&ensp;2. 이론적 방법<br/>
1. 알고리즘을 의사코드(pseudocode)로 표현
2. 원시연산 개수 계산
3. 입력 크기 n의 함수로 표현

# 의사코드

&ensp;알고리즘을 설명하기 위한 사람이 읽기 쉬운 코드<br/>

&ensp;특징<br/>
* 실제 프로그램 언어 아님
* 알고리즘 구조만 표현
* 구현 세부사항 제외

&ensp;예<br/>
```
arrayMax(A, n)
currentMax ← A[0]

for i ← 1 to n-1
    if A[i] > currentMax
        currentMax ← A[i]

return currentMax
```

&ensp;배열에서 최대값 찾는 알고리즘<br/>

# RAM 모델 (Random Access Machine)

&ensp;알고리즘 분석에서 사용하는 가상의 컴퓨터 모델<br/>
&ensp;특징<br/>
* CPU 하나
* 무한 메모리
* 메모리 접근 시간 동일

```
모든 기본 연산 = 같은 시간
```

# 원시 시간(Primitive Operation)

&ensp;알고리즘의 기본 연산<br/>
| 연산    | 설명      |
| ----- | ------- |
| 산술연산  | + - * / |
| 비교    | > <     |
| 대입    | =       |
| 배열 접근 | A[i]    |
| 함수 호출 | call    |


&ensp;RAM 모델에서는 원시연산 = 상수시간<br/>

# Big-O 표기법

&ensp;알고리즘 실행시간의 증가율을 나타내는 방법<br/>
&ensp;정의<br/>
```
f(n) = O(g(n))
```

&ensp;의 의미<br/>
&ensp;f(n)의 증가 속도가 g(n)보다 빠르지 않다.<br/>

&ensp;즉<br/>
```
f(n) ≤ c × g(n)
```

&ensp;가 되는 상수 c 존재<br/>

&ensp;예시 1<br/>
```
2n + 10 = O(n)
```

&ensp;이유<br/>
```
2n + 10 ≤ 3n
```

&ensp;예시 2<br/>
```
3n³ + 20n² + 5 = O(n³)
```

&ensp;Big-O 계산 규칙
1. 최고차항만 남김
2. 상수 제거

## Big-O 계산 규칙

&ensp;규칙 1: 상수 제거<br/>
&ensp;예: 5n → O(n)<br/>

&ensp;규칙 2: 낮은 차수 제거<br/>
&ensp;예: n² + n + 10 → O(n²)<br/>

## 반복문 분석

&ensp;단일 반복문<br/>
```code
for i = 1 to n
```

&ensp;시간 복잡도<br/>
&ensp;O(n)<br/>

&ensp;중첩 반복문<br/>
```code
for i = 1 to n
   for j = 1 to n
```

&ensp;시간복잡도<br/>
&ensp;O(n²)<br/>

&ensp;세 번 중첩<br>
&ensp;O(n³)<br/>

## 조건문 분석

```
if 조건
   A
else
   B
```

&ensp;시간<br/>
&ensp;O(max(A, B))<br/>

# Big-O 친척들

&ensp;1. Big-O<br/>
&ensp;상한<br/>
```
f(n) ≤ g(n)
```

&ensp;2. Big-Omega<br/>
&ensp;하한<br/>
```
f(n) ≥ g(n)
```

&ensp;Big-Theta<br/>
&ensp;정확한 증가율: `Θ(n)`<br/>
&ensp;조건: `O(n) 이면서 Ω(n)` <br/>

&ensp;대표적인 증가율<br/>
&ensp;작은 -> 큰<br/>
```
O(1)
O(log n)
O(n)
O(n log n)
O(n²)
O(n³)
O(2ⁿ)
```

&ensp;성능 비교<br/>
```
좋음     나쁨
O(1) → O(2ⁿ)
```

## 예제: 행렬에서 값 찾기

&ensp;행렬 크기<br/>
&ensp;`n × n`<br/>

&ensp;최악 경우<br/>
&ensp;`n² 번 검사` <br/>

&ensp;그래서<br/>
&ensp;`O(n²)` <br/>

## Prefix Average 예제
&ensp;배열 `X` 가 있을 때<br/>
&ensp;누적 평균 배열 `A` 는<br/>
```cpp
A[0] = 2 / 1 = 2
A[1] = (2+4) / 2 = 3
A[2] = (2+4+6) / 3 = 4
A[3] = (2+4+6+8) / 4 = 5
```

&ensp;결과<br/>
```
A = [2, 3, 4, 5]
```

&ensp;수식<br/>

$A[i] = \frac{X[0] + X[1] + .. + X[i]}{i+1}$

### 방법1: 정의 그대로 계산 (O(n²))
&ensp;알고리즘 아이디어: 매번 처음부터 다시 더함<br/>

&ensp;예<br/>
```
A[0] → X[0]
A[1] → X[0] + X[1]
A[2] → X[0] + X[1] + X[2]
```

```cpp
#include <stdio.h>

void prefixAverages1(int X[], double A[], int n) {

    for(int i = 0; i < n; i++) {

        int sum = 0;

        for(int j = 0; j <= i; j++) {
            sum += X[j];
        }

        A[i] = (double)sum / (i + 1);
    }
}

int main() {

    int X[] = {2,4,6,8};
    int n = 4;
    double A[4];

    prefixAverages1(X, A, n);

    for(int i=0;i<n;i++)
        printf("%.2f ", A[i]);

}
```

&ensp;출력<br/>
```
2.00 3.00 4.00 5.00
```

&ensp;실행 과정<br/>
```
i = 0
sum = X[0]

i = 1
sum = X[0] + X[1]

i = 2
sum = X[0] + X[1] + X[2]

i = 3
sum = X[0] + X[1] + X[2] + X[3]
```

&ensp;시간복잡도 분석<br/>
&ensp;반복 횟수<br/>
```math
1 + 2 + 3 + ... + n
```

&ensp;수학식<br/>

$\frac{n(n+1)}{2}$

&ensp;결과<br/>
```
O(n²)
```

### 방법2: 누적합 사용 (O(n))

&ensp;핵심 아이디어: 이미 계산한 합을 저장해서 재사용<br/>

&ensp;예<br/>
```cpp
sum = 0

i=0
sum = 2

i=1
sum = 2 + 4

i=2
sum = 6 + 6
```

&ensp;즉 `sum += X[i]` <br/>

```cpp
#include <stdio.h>

void prefixAverages2(int X[], double A[], int n) {

    int sum = 0;

    for(int i = 0; i < n; i++) {

        sum += X[i];

        A[i] = (double)sum / (i + 1);
    }
}

int main() {

    int X[] = {2,4,6,8};
    int n = 4;
    double A[4];

    prefixAverages2(X, A, n);

    for(int i=0;i<n;i++)
        printf("%.2f ", A[i]);

}
```

&ensp;출력<br/>
```
2.00 3.00 4.00 5.00
```

&ensp;실행 과정<br/>
```cpp
sum = 0

i=0
sum = 2
A[0] = 2

i=1
sum = 6
A[1] = 3

i=2
sum = 12
A[2] = 4

i=3
sum = 20
A[3] = 5
```

&ensp;시간 복잡도<br/>
&ensp;반복문<br/>
```cpp
for i = 0 → n
```

&ensp;한 번만 반복 (`O(n)`) <br/>

&ensp;두 알고리즘 비교<br/>
| 알고리즘            | 방법           | 시간복잡도 |
| --------------- | ------------ | ----- |
| prefixAverages1 | 매번 처음부터 합 계산 | O(n²) |
| prefixAverages2 | 누적합 저장       | O(n)  |
